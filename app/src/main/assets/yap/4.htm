Объекты, используемые в программах, хранятся в оперативной памяти по определенным адресам и могут занимать разное количество байтов. Есть несколько вариантов выделения памяти для объектов:

по времени выделения – статический и динамический. Статические объекты получают память в самом начале работы программы и освобождают по окончании работы. Динамические получают и освобождают память во время работы по мере необходимости.

По способу выделения – автоматический и управляемый программистом. Автоматические объекты получают память при входе в соответствующий блок программы и освобождают при выходе из него. Управляемое программистом выделение и освобождение памяти выполняется с помощью специальных операций или функций.

Автоматические объекты описываются внутри блоков (тел функций) с описателем auto и область их видимости в программе - это фрагмент от описания до конца блока. Кроме того, к автоматическим объектам относятся все формальные параметры функций. Вся память для автоматических объектов выделяется в адресном пространстве данного приложения (программы).

В специальном участке памяти ОС хранится таблица динамической памяти, которая выделяется и освобождается по требованию программиста в разных приложениях. Операционная система следит за правильностью выделения и освобождения памяти, а при нехватке оперативной памяти для активных приложений может осуществлять своппинг (SWAPPING) с диском.

 Для динамического выделения памяти используется операция new , в которой необходимо указать для какого типа объектов выделяется память и количество объектов. При успешном выполнении запроса эта операция возвращает адрес выделенной памяти, в противном случае – пустой адрес NULL.

Освобождение памяти выполняется операцией delete, в которой указывается адрес ранее полученной памяти. К ошибкам будут приводить попытки освобождения памяти, полученной другим механизмом, например статической или автоматической. Также запрещается память, полученную одним куском, освобождать отдельными операциями или освобождать память с другого адреса.

Размер оперативной памяти доступной, для динамического выделения, зависит от конфигурации используемого компьютера и возможностей операционной системы.

Имеется несколько функций для работы с динамической памятью, которые можно использовать в разных операционных системах, но они могут иметь свои специфичные особенности при выполнении.

int coreleft () возвращает беззнаковое целое число - размер доступной памяти для выделения в байтах.

char * malloc (long int) пытается выделить требуемое количество памяти и возвращает адрес начала участка. Если памяти недостаточно, то возвращает пустой адрес.

void free (char *) освобождает ранее выделенный участок памяти и возвращает его в стек доступной памяти для дальнейшего использования.
