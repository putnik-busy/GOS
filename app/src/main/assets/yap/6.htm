Препроцессор представляет собой разновидность транслятора со специализированного языка работающего в режиме интерпретации, т.е. он распознает свои операторы в тексте и тут же их выполняет. В качестве входных данных препроцессор получает программу на языке препроцессора, а в результате должна получиться программа на языке программирования, например Си++ или Паскале.

Для внесения изменений в исходный текст используются переменные препроцессора, которые имеют какие-то значения и могут быть активными для замены или неактивными.

При своей работе препроцессор последовательно сканирует (просматривает) исходный текст. Если встречается оператор препроцессора, то он выполняется, если встречается активная препроцессорная переменная (ее имя), то вместо нее в текст вставляется ее значение. Сканирование продолжается с начала вставленного фрагмента, потому, что в нем также могут быть препроцессорные переменные.

Основной оператор препроцессора, который позволяет компоновать текст, оператор вставки из файла имеет вид: # include “имя-файла”.

Препроцессорные переменные бывают только символьного типа, т.е. их значения рассматриваются как символьные строки, но при записи они ограничиваются только пробелом в начале строки и концом строки текста.

Описание препроцессорных переменных и присвоение им значений выполняется оператором: # define ИМЯ ВЫРАЖЕНИЕ. Заданное выражение в символьной форме является значением переменной и она становится активной. Использование имени препроцессорной переменной в операторе отмены # undef делает ее неактивной.

Стандартная функция defined (<имя ПП-переменной>) возвращает истину, если указанная препроцессорная переменная уже определена.

Последовательное сканирование текста можно изменить на условное, :

# if <Условное выражение>

// Фрагмент сканируется, если условие истинно

# else

// Фрагмент сканируется, если условие ложно

# endif

Для внесения в текст изменений, зависящих от контекста, в языке препроцессора предусмотрены макросы – аналоги функций с параметрами из языков программирования. Например, макрос

будет заменяться в тексте в зависимости от значений параметров x и y, с которыми его используют. Использование макросов вместо обычных функций иногда может привести к увеличению размера загрузочной программы, но дать экономию во времени ее выполнения.
